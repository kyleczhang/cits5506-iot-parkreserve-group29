[LED Command String Reference (Pi → ESP32)]
================================================================
State                   LED Command String        ESP32 Behavior
available             → "available"           → green light steady
reserved              → "reserved"            → yellow light steady
pending_check_in      → "pending_check_in"    → yellow light blinking
reserved_checked_in   → "reserved_checked_in" → red light steady
occupied (no booking) → "reserved_checked_in" → red light steady
conflict              → "conflict_strong"     → red light blinking + buzzer

================================================================
[Sensor Message Format (ESP32 → Pi, both supported)]
================================================================
# Plain string format (what ESP32 actually sends)
mosquitto_pub -h localhost -t "bay/1/status" -m "occupied"
mosquitto_pub -h localhost -t "bay/1/status" -m "available"

# JSON format (compatible for local testing)
mosquitto_pub -h localhost -t "bay/A1/status" -m '{"occupied":true,"distance_cm":12.0}'
mosquitto_pub -h localhost -t "bay/A1/status" -m '{"occupied":false,"distance_cm":280.0}'

  2. mosquitto_pub -h localhost -t "bay/A1/status" -m "occupied"
  3. curl -X POST "http://localhost:5000/api/v1/bays/A1/image" \
       -H "Content-Type: image/jpeg" \
       -H "X-API-Key: ParkReserve-Group29-SuperSecret" \
       -H "X-Timestamp: 20260513_100000" \
       --data-binary "@/Users/congyuan/Desktop/plate.jpg"

Expected Backend messages in order:

  {"state":"pending_check_in","last_distance_cm":0.0,"ts":"...","event_id":"..."}

  // cloud/bay/A1/event
  {"event":"pending_check_in","ts":"...","event_id":"...","reservation_id":"550e8400-..."}

  {"state":"reserved_checked_in","last_distance_cm":0.0,"ts":"...","event_id":"..."}

Scenario C: Wrong vehicle leaves after conflict_strong → Backend restores reservation
Steps:
  1. Complete Scenario B (enter conflict state)
  2. mosquitto_pub -h localhost -t "bay/A1/status" -m "available"

Expected Pi log:
  [A1] Vehicle left after conflict, reporting available, awaiting Backend decision
  [A1] conflict → available
  [LocalMQTT] → bay/A1/led  'available'

Expected Backend message:
  // cloud/bay/A1/state
  {"state":"available","last_distance_cm":0.0,"ts":"...","event_id":"..."}

After Backend processes and re-sends create, Pi receives it:
  [A1] Restoring reservation cache during conflict, waiting for vehicle to leave to return to reserved
  (currently available state; after receiving create it transitions to reserved)

Scenario D: LPR confidence too low → manual check-in timeout → conflict (weak)
Preparation: change mock_confidence in control_service.py
  self.mock_confidence = 0.50   # below 0.80 threshold
  and change MANUAL_CHECKIN_GRACE_SECONDS in settings.py to 30 (for testing)

Steps:
  1. Send reservation
  2. mosquitto_pub -h localhost -t "bay/A1/status" -m "occupied"
  3. curl upload image (LPR confidence too low, stays at pending_check_in)
  4. Wait 30 seconds for timeout

Expected Pi log (after timeout):
  [A1] Manual check-in timeout → conflict (Backend handles conflict_weak internally)
  [LocalMQTT] → bay/A1/led  'conflict_strong'

Expected Backend message (state only, no event):
  // cloud/bay/A1/state
  {"state":"conflict","last_distance_cm":0.0,"ts":"...","event_id":"..."}

[3. Interfaces for Receiving ESP32 Messages]
================================================================

Interface 1: Sensor status (sent every 2 seconds)
Topic: bay/<bay_id>/status
ESP32 send format (plain string):
  "occupied"   → vehicle present
  "available"  → no vehicle

Expected Pi behavior:
  with reservation:    "occupied" → pending_check_in
  without reservation: "occupied" → occupied
  "available" → depending on context → reserved or available

---

Interface 2: License plate image upload (triggered once when vehicle arrives)
Endpoint: POST http://<pi_ip>:5000/api/v1/bays/<bay_id>/image
Headers:
  Content-Type: image/jpeg
  X-API-Key: ParkReserve-Group29-SuperSecret
  X-Timestamp: 20260513_100000  (optional)
Body: raw binary JPEG

Test command:
curl -X POST "http://localhost:5000/api/v1/bays/A1/image" \
  -H "Content-Type: image/jpeg" \
  -H "X-API-Key: ParkReserve-Group29-SuperSecret" \
  -H "X-Timestamp: 20260513_100000" \
  --data-binary "@/Users/congyuan/Desktop/plate.jpg"

Expected Pi response (202):
  {"message": "Image securely received", "file": "bay_A1_20260513_100000.jpg"}

Expected Pi log:
  [ImageReceiver] Saved: ./tmp/parkReserve/images/bay_A1_20260513_100000.jpg
  [LPR] EasyOCR raw results: [...]
